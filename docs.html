Technische Architektur

shared.js & style.css

Alle Seiten teilen sich shared.js und (außer index.html) style.css. Diese Dateien deklarieren globale Variablen — niemals in Inline-Scripts neu deklarieren.

Was shared.js global bereitstellt

const

nav — Referenz auf #main-nav. Auf Unterseiten stattdessen slideHero für Nav-Scroll verwenden.

const

lb, lbImg — Lightbox DOM-Element. Niemals erneut document.createElement('div') für Lightbox aufrufen.

fn

openLightbox(src) — Lightbox öffnen mit Bild-URL. Direkter Aufruf in Gallery-Seiten.

fn

openMenu() / closeMenu() — Mobile Menu öffnen/schließen.

fn

setLang(lang) — Sprache wechseln. Liest data-i18n und data-i18n-ph Attribute.

obs

revealObserver — IntersectionObserver für .reveal Klasse. Fügt visible hinzu wenn im Viewport.

index.html — Sonderregel

index.html lädt kein style.css. Alle CSS-Regeln sind inline im <style>-Block. Alle anderen Seiten laden:

<link rel="stylesheet" href="style.css?v=4" />
<style>
  /* seitenspezifische Styles */
  .nav-logo { ... }
  .SEITE-grid { ... }
</style>

  break-inside: avoid;
  margin-bottom: 6px;
  overflow: hidden;
  cursor: zoom-in;

  width: 100%;
  height: auto; /* ← natürliche Bildhöhe! */

JS Pattern

Flash Slideshow

Hero-Slideshow mit 1-Sek-Intervall und weißem Flash-Effekt beim Bildwechsel. Zwei Slides (A/B) wechseln sich ab.

// Flash-Transition Slideshow
const shA    = document.getElementById('sh-a');
const shB    = document.getElementById('sh-b');
const shFlash = document.getElementById('sh-flash');
let shAktiv = shA, shNaechst = shB, shIdx = 0;

function shShow(src) {
  shNaechst.style.backgroundImage =
    `url('img/ORDNER/${encodeURIComponent(src)}')`;
  // Flash: sofort weiß
  shFlash.style.transition = 'none';
  shFlash.style.opacity = '1';
  // Slides tauschen
  shAktiv.classList.remove('active');
  shNaechst.classList.add('active');
  [shAktiv, shNaechst] = [shNaechst, shAktiv];
  // Flash ausblenden
  requestAnimationFrame(() => requestAnimationFrame(() => {
    shFlash.style.transition = 'opacity 0.25s ease';
    shFlash.style.opacity = '0';
  }));
}

shShow(FILES[0]);
setInterval(() => {
  shIdx = (shIdx + 1) % FILES.length;
  shShow(FILES[shIdx]);
}, 1000);

setLang() Funktion

function setLang(lang) {
  currentLang = lang;
  localStorage.setItem('db_lang', lang);
  const t = i18n[lang];

  // Text-Inhalte
  document.querySelectorAll('[data-i18n]')
    .forEach(el => {
      const key = el.getAttribute('data-i18n');
      if (t[key] !== undefined) el.innerHTML = t[key];
    });

  // Placeholders
  document.querySelectorAll('[data-i18n-ph]')
    .forEach(el => {
      const key = el.getAttribute('data-i18n-ph');
      if (t[key] !== undefined) el.placeholder = t[key];
    });

  // Aktiver Button
  document.querySelectorAll('.lang-btn')
    .forEach(btn => {
      btn.classList.toggle('active',
        btn.dataset.lang === lang);
    });

  document.documentElement.lang = lang;
}

// Beim Laden automatisch Sprache setzen:
document.addEventListener('DOMContentLoaded',
  () => setLang(currentLang));

/webbilder Claude Command

Optimiert alle Fotos in einem Ordner für die Webseite. Speichert optimierte Versionen in einem web/ Unterordner. Originale bleiben unverändert.

Was der Skill macht — Schritte

1

2

Unterordner erstellen

3

Alle Fotos optimieren

sips skaliert auf max. 2000px, Qualität 80%, Format JPEG. Jedes Foto zeigt Vorher/Nachher-Größe.

4

Optional: in Portfolio kopieren

Am Ende wird gefragt ob die Fotos direkt in den db Portfolio/img/ Ordner kopiert werden sollen.

Der sips Befehl

mkdir -p "$ORDNER/web"

for f in "$ORDNER"/*.{jpg,jpeg,JPG,JPEG,png,PNG}; do
  [ -f "$f" ] || continue
  name=$(basename "$f")
  namejpg="${name%.*}.jpg"

  sips -Z 2000 \
       -s format jpeg \
       -s formatOptions 80 \
       "$f" --out "$ORDNER/web/$namejpg" 2>/dev/null

  orig=$(du -sh "$f" | cut -f1)
  new=$(du -sh "$ORDNER/web/$namejpg" | cut -f1)
  echo "$name: $orig → $new"
done

Security Checkliste

GitHub Sicherheit
absichern

Checkliste — noch ausstehend

○

1. Starkes Passwort setzen
github.com → Settings → Password and Authentication → Change Password.
Mindestens 16 Zeichen, Kombination aus Groß/Klein/Zahlen/Symbolen. Am besten Passwort-Manager (1Password, Bitwarden).

○

2. Zwei-Faktor-Authentifizierung (2FA) aktivieren
Settings → Password and Authentication → Enable two-factor authentication.
Empfohlen: Authenticator App (Google Authenticator, Authy oder 1Password).
NICHT nur SMS — kann gehackt werden (SIM Swapping).

○

3. Backup-Codes sichern
Nach 2FA-Aktivierung zeigt GitHub 16 Backup-Codes. Diese an einem sicheren Ort speichern (ausdrucken oder Passwort-Manager).
Diese Codes erlauben Login wenn Handy verloren/kaputt.

○

4. SSH Key statt Passwort für Git Push
Settings → SSH and GPG keys → New SSH Key.
Terminal: ssh-keygen -t ed25519 -C "dubobulic@gmail.com"
Public Key (~/.ssh/id_ed25519.pub) in GitHub eintragen.

○

5. Aktive Sessions prüfen
Settings → Sessions → Alle unbekannten Geräte abmelden.
Settings → Security log → Prüfen ob verdächtige Logins vorhanden.

○

6. E-Mail-Adresse bestätigen
Settings → Emails → Sicherstellen dass dubobulic@gmail.com verifiziert ist und Backup-E-Mail gesetzt ist.

○

7. Repository-Sichtbarkeit prüfen
Für deploy-Repo (dubobulic.com): kann Public sein (statische HTML).
Kein API-Keys, Passwörter oder .env Dateien in Repos committen!

○

8. Vercel Account absichern
vercel.com → Account Settings → Security → 2FA aktivieren.
Vercel mit GitHub verknüpft = wenn GitHub kompromittiert, auch Vercel gefährdet.

SSH Key einrichten — Kurzanleitung

# 1. Key generieren:
ssh-keygen -t ed25519 -C "dubobulic@gmail.com"
# → Speichern in ~/.ssh/id_ed25519
# → Passphrase optional (empfohlen)

# 2. Public Key anzeigen:
cat ~/.ssh/id_ed25519.pub
# → Diesen Text komplett kopieren

# 3. In GitHub eintragen:
# Settings → SSH Keys → New SSH Key
# Title: "Mac Mini 2025"
# Key: (Public Key einfügen)

# 4. Remote URL ändern:
git remote set-url origin
  git@github.com:DuboB/db-portfolio.git

# 5. Testen:
ssh -T git@github.com
# → "Hi DuboB! You've successfully authenticated"
